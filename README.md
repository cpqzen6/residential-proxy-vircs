## 住宅代理 vs. 数据中心 IP：做数据采集，差距在哪里

先把概念说清楚，后面看 VIRCS 才有参照。

**数据中心 IP**：来自云服务商或机房，响应快、价格低，但 IP 段是公开的数据中心段，反爬工具一眼识别。Amazon、Google、TikTok 这类平台对机房 IP 审查很严，触发风控的概率高。

**共享住宅代理池**：汇聚大量普通用户贡献的 IP，按 GB 计费，IP 数量多、覆盖范围广，但 IP 被很多人共用，脏 IP 混入的概率不低，而且 IP 来源质量参差不齐。

**静态住宅 IP（VIRCS 的模式）**：直接从 AT&T 这类大型 ISP 申请住宅宽带，IP 归属显示为真实家庭地址，独享不共用，不存在"别人把这个 IP 用烂了"的问题。缺点是每个 IP 背后有一套真实的物理线路，成本高，所以价格也高。

用于美国公开数据采集时，三类 IP 的表现差异很直接：

| 类型 | 数据中心 IP | 共享住宅代理池 | 静态住宅 IP（VIRCS）|
|---|---|---|---|
| 被识别风险 | 高 | 中 | 低 |
| IP 稳定性 | 高 | 波动（依赖用户在线）| 高（专线）|
| 适合高风控平台 | 不适合 | 部分适合 | 适合 |
| 费用模型 | 按 IP 或按月 | 按 GB | 按月 |
| IP 独占性 | 无 | 无 | 独享 |
